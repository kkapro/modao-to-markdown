# modao-to-markdown

将墨刀（Mockplus）原型页面采集并转换为结构化 Markdown 文档的工具，支持 AI 图像理解生成可检索的需求文档。

## 功能概述

本工具解决以下问题：

- 产品经理提供墨刀链接，但 AI 无法直接读取
- 墨刀导出的图片难以被检索和复用
- 手动整理需求文档效率低

通过两阶段处理流程：
1. **页面采集** - 自动遍历墨刀页面，截图 + 提取文字
2. **AI 结构化** - 调用视觉大模型将截图转换为结构化 Markdown

## 目录结构

```
modao-to-markdown/
├── save_modao.py      # 墨刀页面采集脚本
├── image_to_md.py    # AI 结构化转换脚本
└── README.md
```

## 环境要求

- Python 3.8+
- Playwright (`pip install playwright`)
- OpenAI SDK (`pip install openai`)
- Pillow (`pip install pillow`)
- 阿里百炼 API Key（或兼容的 OpenAI API）

## 安装

```bash
pip install openai pillow playwright numpy scipy
playwright install chromium
```

## 使用方式

### 阶段一：采集墨刀页面

```bash
python save_modao.py "墨刀项目URL"
```

参数：
- 第1个参数：墨刀项目 URL（必填）
- 第2个参数：画布索引（可选，如 `1` 表示只采集第1个画布）
- 第3个参数：`debug`（可选，覆盖已有文件）

输出目录：`modao-export/项目名/`

### 阶段二：AI 结构化转换

```bash
python image_to_md.py "项目目录"
```

参数：
- 第1个参数：项目根目录（必填）
- 第2个参数：`debug`（可选，覆盖已有文件）

目录结构要求：
```
项目目录/
├── images/           # 截图文件
└── ai-md/           # 输出的 MD 文档（自动生成）
```

## 配置 API Key

在 `image_to_md.py` 中修改：

```python
BAILIANT_API_KEY = "your-api-key-here"  # 替换为你的 API Key
BAILIANT_MODEL = "qwen3-vl-flash"        # 或其他视觉大模型
```

支持的模型：
- 阿里百炼 qwen3-vl-flash
- OpenAI GPT-4V（需修改 base_url）
- 其他兼容 OpenAI API 的视觉模型
